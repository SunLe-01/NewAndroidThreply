# B 模式回复卡片快速操作指南

## 问题解决

**问题**：看不到 B 模式的回复卡片 UI

**原因**：
1. AI 面板默认隐藏
2. 需要输入文本
3. 需要点击按钮生成回复

**解决方案**：已优化 UI，现在点击"B"按钮会自动生成回复

---

## 快速操作（3 步）

### 步骤 1：输入文本
在键盘输入框中输入任何文本，例如：
```
你好，今天天气真好
```

### 步骤 2：打开 AI 面板
点击键盘上方的任何技能卡片（翻译/替换/润色），AI 面板会显示

### 步骤 3：点击"B"按钮生成回复
在 AI 面板顶部点击"B"按钮，系统会：
1. 自动切换到 B 模式
2. 自动生成 3 条回复建议
3. 显示为 3 张卡片

---

## 看到回复卡片后的操作

### 点击回复卡片
- 提交该回复到输入框

### 长按回复卡片
- 生成 3 条子回复
- 显示"← 返回"按钮
- 可以继续长按子回复展开更多层级

### 点击"← 返回"
- 返回上一层回复

### 点击"刷新"
- 重新生成回复

### 点击"退出"
- 关闭 AI 面板

---

## 完整示例

```
1. 输入：你好，今天天气真好

2. 点击"翻译"卡片 → AI 面板打开

3. 点击"B"按钮 → 自动生成 3 条回复：
   ┌─────────────────────────────┐
   │ 是啊，天气很好呢！         │
   └─────────────────────────────┘
   ┌─────────────────────────────┐
   │ 确实，今天阳光明媚         │
   └─────────────────────────────┘
   ┌─────────────────────────────┐
   │ 对，适合出去走走           │
   └─────────────────────────────┘

4. 长按第一张卡片 → 生成子回复：
   ┌─────────────────────────────┐
   │ ← 返回                      │
   ├─────────────────────────────┤
   │ 是啊，我们一起去公园吧     │
   ├─────────────────────────────┤
   │ 天气这么好，不出去太可惜了 │
   ├─────────────────────────────┤
   │ 对，我也想出去透透气       │
   └─────────────────────────────┘

5. 点击子回复 → 提交文本到输入框

6. 点击"← 返回" → 返回上一层
```

## 故障排除

### 问题：点击"B"按钮没有反应

**检查清单**：
1. ✅ 是否输入了文本？
2. ✅ 是否打开了 AI 面板？
3. ✅ 是否有网络连接？
4. ✅ 是否正在加载中？（等待几秒）

### 问题：看不到"B"按钮

**解决方案**：
1. 点击任何技能卡片（翻译/替换/润色）打开 AI 面板
2. "B"按钮会在 AI 面板顶部显示

### 问题：长按卡片没有反应

**检查清单**：
1. ✅ 长按时间是否足够长？（需要 500ms 以上）
2. ✅ 卡片是否为空？
3. ✅ 是否正在加载中？

### 问题：生成的回复为空

**可能原因**：
1. 输入文本太短或不清楚
2. API 连接失败
3. 网络超时

**解决方案**：
1. 输入更长、更清楚的文本
2. 检查网络连接
3. 点击"刷新"重试

## 技术细节

### 数据流
```
输入文本
  ↓
点击"B"按钮
  ↓
setAiMode(ImeAiMode.B) + triggerAiFromCurrentInput()
  ↓
AiImeCoordinator.requestSuggestions()
  ↓
调用 AI API
  ↓
流式接收回复
  ↓
更新 SuggestionState
  ↓
BModePanel 显示 3 张卡片
```

### 长按流程
```
长按卡片
  ↓
detectTapGestures 检测长按（500ms）
  ↓
onLongPress 回调
  ↓
AiImeCoordinator.expandReplyForChildren()
  ↓
生成子回复
  ↓
更新 suggestions 的 children 列表
  ↓
显示子回复 + "← 返回"按钮
```
